## Prerequisites

Before you begin, make sure you have:

- **Node.js 20.x** or higher
- **PostgreSQL 14** or higher
- **npm** (comes with Node.js)

## Setup

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/JhojanRop/scheduly-api.git
    cd scheduly-api
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the project root:

    ```bash .env
    DATABASE_URL="postgresql://USER:PASSWORD@localhost:5432/scheduly_db?schema=public"
    JWT_SECRET="your_jwt_secret_key_here"
    NODE_ENV="development"
    PORT=3000
    ```

    | Variable | Description |
    |----------|-------------|
    | `DATABASE_URL` | PostgreSQL connection string |
    | `JWT_SECRET` | Secret key for signing JWT tokens (use a long random string) |
    | `NODE_ENV` | Set to `development` locally, `production` in prod |
    | `PORT` | Port the server listens on (default: 3000) |

    <Warning>
      Never commit your `.env` file to version control. Add it to `.gitignore`.
    </Warning>
  </Step>

  <Step title="Set up the database">
    Create the database and run migrations:

    ```bash
    npx prisma migrate dev
    ```

    Seed the initial data (days of the week, 1–7):

    ```bash
    npx prisma db seed
    ```

    <Note>
      The seed populates the `days` table with Monday (1) through Sunday (7). This data is required for sections to have meeting days.
    </Note>
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run start:dev
    ```

    The API will be available at `http://localhost:3000/api`.

    <Tip>
      The dev server uses hot reload — changes to TypeScript files automatically restart the server.
    </Tip>
  </Step>

  <Step title="Register your first user">
    ```bash
    curl -X POST http://localhost:3000/api/auth/register \
      -H "Content-Type: application/json" \
      -d '{
        "username": "jhojanRop",
        "password": "12345678",
        "fullName": "Jhojan Ropero"
      }'
    ```

    Expected response:

    ```json
    {
      "id": "a3f4b2c1-...",
      "username": "jhojanRop",
      "fullName": "Jhojan Ropero",
      "createdAt": "2024-01-15T10:00:00.000Z"
    }
    ```
  </Step>

  <Step title="Log in and get your access token">
    ```bash
    curl -X POST http://localhost:3000/api/auth/login \
      -H "Content-Type: application/json" \
      -c cookies.txt \
      -d '{
        "username": "jhojanRop",
        "password": "12345678"
      }'
    ```

    Expected response:

    ```json
    {
      "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
    }
    ```

    Use the access token in all subsequent requests:

    ```bash
    curl http://localhost:3000/api/subjects \
      -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
    ```
  </Step>
</Steps>

## Explore the API

With the server running, visit the Swagger documentation at:

```
http://localhost:3000/docs
```

Swagger provides an interactive explorer where you can try every endpoint directly from the browser.
