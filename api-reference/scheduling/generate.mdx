---
title: 'Generate schedules'
description: 'Generate up to 3 optimized schedule combinations for a set of subjects using your configured rules.'
---

`POST /api/scheduling/generate`

Runs the scheduling engine against the sections belonging to the given subjects. The engine works in two phases:

1. **Pruning** — sections that violate hard rules (e.g. `AVOID_PROFESSOR`, `LUNCH_BREAK_PROTECTED`) are discarded before any combinations are explored.
2. **Backtracking + scoring** — the engine explores all conflict-free combinations via backtracking, scores each one against your ordered rules, and returns the top 3 results ordered by score descending.

<Note>
  You must have at least one rule configured before calling this endpoint, and every subject you pass must have at least one section. If a subject has no sections, or if pruning removes all sections for a subject, the request will fail with `400 Bad Request`.
</Note>

## Authentication

Requires a valid Bearer token in the `Authorization` header.

<ParamField header="Authorization" type="string" required>
  Bearer token obtained from the login endpoint. Format: `Bearer <token>`.
</ParamField>

## Request body

<ParamField body="subjectIds" type="string[]" required>
  Array of subject UUIDs to include in the generated schedules. Must contain at least one entry. Every UUID must correspond to a subject that belongs to the authenticated user and has at least one section.
</ParamField>

## Response

Returns an array of up to 3 schedule objects ordered by `score` descending (best match first).

<ResponseField name="[].rank" type="number" required>
  Position of this schedule in the results (1 = best score).
</ResponseField>

<ResponseField name="[].score" type="number" required>
  Composite score calculated from your rules and their priority order. Higher is better. Rounded to two decimal places.
</ResponseField>

<ResponseField name="[].sections" type="object[]" required>
  One entry per subject, representing the section assigned to that subject in this schedule combination.

  <Expandable title="section properties">
    <ResponseField name="sectionId" type="string" required>
      UUID of the section.
    </ResponseField>
    <ResponseField name="subject" type="string" required>
      Name of the subject this section belongs to.
    </ResponseField>
    <ResponseField name="professor" type="string | null" required>
      Full name of the professor assigned to the section, or `null` if no professor is assigned.
    </ResponseField>
    <ResponseField name="startTime" type="string" required>
      Section start time in `HH:MM` format (24-hour, UTC).
    </ResponseField>
    <ResponseField name="endTime" type="string" required>
      Section end time in `HH:MM` format (24-hour, UTC).
    </ResponseField>
    <ResponseField name="days" type="string[]" required>
      Days of the week on which the section meets (e.g. `["monday", "wednesday"]`).
    </ResponseField>
  </Expandable>
</ResponseField>

## Error codes

| Status | Description |
| ------ | ----------- |
| `400`  | No sections found for one or more of the provided subject IDs. |
| `400`  | Pruning removed all sections for a subject — consider relaxing or removing some rules. |
| `400`  | No conflict-free combinations could be found given the available sections. |
| `401`  | Missing or invalid Bearer token. |

```bash cURL
  --header 'Content-Type: application/json' \
  --data '{
    "subjectIds": [

      "b2c3d4e5-f6a7-8901-bcde-f12345678901",
      "c3d4e5f6-a7b8-9012-cdef-123456789012"
    ]
  }'
```
