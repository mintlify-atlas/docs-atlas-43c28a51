---

`POST /api/professors`

Creates a new professor associated with the authenticated user's account.

## Authentication

This endpoint requires a valid Bearer token in the `Authorization` header.

## Request

### Body parameters

<ParamField body="fullName" type="string" required>
  The full name of the professor. Maximum 100 characters.
</ParamField>

## Response

Returns the created professor object.

<ResponseField name="id" type="string" required>
  Unique identifier for the professor (UUID).
</ResponseField>

<ResponseField name="fullName" type="string" required>
  The full name of the professor.
</ResponseField>

<ResponseField name="userId" type="string">
  The ID of the user who owns this professor. May be `null` for system-level records.
</ResponseField>

## Error codes

| Status | Description |
| ------ | ----------- |
| `401 Unauthorized` | Missing or invalid Bearer token. |
| `400 Bad Request` | Request body is missing required fields or fails validation. |

<RequestExample>

```bash cURL
curl --request POST \
  --url https://api.scheduly.app/api/professors \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "fullName": "Juan Pérez"
  }'
```

</RequestExample>

<ResponseExample>

```json 201
{
  "id": "c3d4e5f6-a7b8-9012-cdef-123456789012",
  "fullName": "Juan Pérez",
  "userId": "f1e2d3c4-b5a6-7890-fedc-ba9876543210"
}
```

</ResponseExample>
