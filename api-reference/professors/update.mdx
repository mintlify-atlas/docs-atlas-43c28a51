---
title: 'Update professor'
description: 'Update the full name of an existing professor.'
---

`PUT /api/professors/:id`

Updates an existing professor. The professor must belong to the authenticated user.

## Authentication

This endpoint requires a valid Bearer token in the `Authorization` header.

## Request

### Path parameters

<ParamField path="id" type="string" required>
  The UUID of the professor to update.
</ParamField>

### Body parameters

<ParamField body="fullName" type="string" required>
  The new full name of the professor. Maximum 100 characters.
</ParamField>

## Response

Returns the updated professor object.

<ResponseField name="id" type="string" required>
  Unique identifier for the professor (UUID).
</ResponseField>

<ResponseField name="fullName" type="string" required>
  The updated full name of the professor.
</ResponseField>

<ResponseField name="userId" type="string">
  The ID of the user who owns this professor. May be `null` for system-level records.
</ResponseField>

## Error codes

| Status | Description |
| ------ | ----------- |
| `401 Unauthorized` | Missing or invalid Bearer token. |
| `400 Bad Request` | Request body is missing required fields or fails validation. |
| `404 Not Found` | No professor with the given ID exists for the authenticated user. |

<RequestExample>

```bash cURL
curl --request PUT \
  --url https://api.scheduly.app/api/professors/c3d4e5f6-a7b8-9012-cdef-123456789012 \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "fullName": "Juan Carlos Pérez"
  }'
```

</RequestExample>

<ResponseExample>

```json 200
{
  "id": "c3d4e5f6-a7b8-9012-cdef-123456789012",
  "fullName": "Juan Carlos Pérez",
  "userId": "f1e2d3c4-b5a6-7890-fedc-ba9876543210"
}
```

```json 404
{
  "statusCode": 404,
  "message": "Professor not found",
  "error": "Not Found"
}
```

</ResponseExample>
