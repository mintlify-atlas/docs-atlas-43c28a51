---
title: 'Get professor'
description: 'Retrieve a single professor by their ID.'
---

`GET /api/professors/:id`

Returns a single professor by ID. The professor must belong to the authenticated user.

## Authentication

This endpoint requires a valid Bearer token in the `Authorization` header.

## Request

### Path parameters

<ParamField path="id" type="string" required>
  The UUID of the professor to retrieve.
</ParamField>

## Response

Returns the professor object.

<ResponseField name="id" type="string" required>
  Unique identifier for the professor (UUID).
</ResponseField>

<ResponseField name="fullName" type="string" required>
  The full name of the professor.
</ResponseField>

<ResponseField name="userId" type="string">
  The ID of the user who owns this professor. May be `null` for system-level records.
</ResponseField>

## Error codes

| Status | Description |
| ------ | ----------- |
| `401 Unauthorized` | Missing or invalid Bearer token. |
| `404 Not Found` | No professor with the given ID exists for the authenticated user. |

<RequestExample>

```bash cURL
curl --request GET \
  --url https://api.scheduly.app/api/professors/c3d4e5f6-a7b8-9012-cdef-123456789012 \
  --header 'Authorization: Bearer <token>'
```

</RequestExample>

<ResponseExample>

```json 200
{
  "id": "c3d4e5f6-a7b8-9012-cdef-123456789012",
  "fullName": "Juan Pérez",
  "userId": "f1e2d3c4-b5a6-7890-fedc-ba9876543210"
}
```

```json 404
{
  "statusCode": 404,
  "message": "Professor not found",
  "error": "Not Found"
}
```

</ResponseExample>
