---
title: 'Create section'
description: 'Create a new section with a time range, subject, optional professor, and meeting days.'
---

`POST /api/sections`

Creates a new section belonging to the authenticated user's subject. The section includes a start and end time, the days it meets, and an optional professor assignment.

<Info>
  All requests must include a valid Bearer token in the `Authorization` header.
</Info>

## Request body

<ParamField body="startTime" type="string" required>
  Section start time in `HH:MM` 24-hour format. Example: `"07:00"`.
</ParamField>

<ParamField body="endTime" type="string" required>
  Section end time in `HH:MM` 24-hour format. Must be after `startTime`. Example: `"09:00"`.
</ParamField>

<ParamField body="subjectId" type="string" required>
  UUID of the subject this section belongs to. The subject must belong to the authenticated user.
</ParamField>

<ParamField body="professorId" type="string">
  UUID of the professor assigned to this section. The professor must belong to the authenticated user. Omit to leave unassigned.
</ParamField>

<ParamField body="dayIds" type="number[]" required>
  Array of day IDs representing the days this section meets. Values range from `1` (Monday) to `7` (Sunday). Example: `[1, 3, 5]` for Monday, Wednesday, Friday.
</ParamField>

## Response

<ResponseField name="id" type="string" required>
  Unique identifier (UUID) of the created section.
</ResponseField>

<ResponseField name="startTime" type="string" required>
  Section start time as an ISO 8601 timestamp (time stored with timezone).
</ResponseField>

<ResponseField name="endTime" type="string" required>
  Section end time as an ISO 8601 timestamp (time stored with timezone).
</ResponseField>

<ResponseField name="subjectId" type="string" required>
  UUID of the associated subject.
</ResponseField>

<ResponseField name="professorId" type="string">
  UUID of the assigned professor, or `null` if unassigned.
</ResponseField>

<ResponseField name="subject" type="object" required>
  <Expandable title="properties">
    <ResponseField name="id" type="string">UUID of the subject.</ResponseField>
    <ResponseField name="name" type="string">Name of the subject.</ResponseField>
    <ResponseField name="userId" type="string">UUID of the owning user.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="professor" type="object">
  <Expandable title="properties">
    <ResponseField name="id" type="string">UUID of the professor.</ResponseField>
    <ResponseField name="fullName" type="string">Full name of the professor.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="sectionDays" type="object[]" required>
  <Expandable title="properties">
    <ResponseField name="id" type="string">UUID of the section-day record.</ResponseField>
    <ResponseField name="sectionId" type="string">UUID of the parent section.</ResponseField>
    <ResponseField name="dayId" type="number">Day identifier (1=Monday … 7=Sunday).</ResponseField>
    <ResponseField name="day" type="object">
      <Expandable title="properties">
        <ResponseField name="id" type="number">Day ID.</ResponseField>
        <ResponseField name="name" type="string">Day name, e.g. `"Monday"`.</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## Error codes

| Status | Description |
| ------ | ----------- |
| `400`  | Validation error — missing or malformed fields. |
| `401`  | Unauthorized — missing or invalid Bearer token. |
| `404`  | Subject or professor not found, or does not belong to the authenticated user. |

<RequestExample>

```bash cURL
curl --request POST \
  --url https://api.scheduly.app/api/sections \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "startTime": "07:00",
    "endTime": "09:00",
    "subjectId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "professorId": "b2c3d4e5-f6a7-8901-bcde-f12345678901",
    "dayIds": [1, 3, 5]
  }'
```
