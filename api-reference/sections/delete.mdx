---
title: 'Delete section'
description: 'Permanently delete a section and its associated meeting days.'
---

`DELETE /api/sections/:id`

Permanently deletes a section and all of its associated `SectionDay` records. The section must belong to a subject owned by the authenticated user. Returns no content on success.

<Warning>
  This action is irreversible. Deleting a section also removes it from any saved schedules that reference it.
</Warning>

<Info>
  All requests must include a valid Bearer token in the `Authorization` header.
</Info>

## Path parameters

<ParamField path="id" type="string" required>
  UUID of the section to delete.
</ParamField>

## Response

Returns `204 No Content` on success with an empty body.

## Error codes

| Status | Description |
| ------ | ----------- |
| `401`  | Unauthorized — missing or invalid Bearer token. |
| `404`  | Section not found, or does not belong to the authenticated user. |

<RequestExample>

```bash cURL
curl --request DELETE \
  --url https://api.scheduly.app/api/sections/c3d4e5f6-a7b8-9012-cdef-123456789012 \
  --header 'Authorization: Bearer <token>'
```

</RequestExample>

<ResponseExample>

```json 204

```

```json 404
{
  "statusCode": 404,
  "message": "Section not found",
  "error": "Not Found"
}
```

</ResponseExample>
