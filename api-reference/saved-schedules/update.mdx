---
title: 'Update a saved schedule'
description: 'Rename a saved schedule. Only the name field can be updated.'
---

`PUT /api/saved-schedules/:id`

Updates the name of an existing saved schedule. The sections associated with the schedule are not modified by this endpoint.

## Authentication

Requires a valid Bearer token in the `Authorization` header.

<ParamField header="Authorization" type="string" required>
  Bearer token obtained from the login endpoint. Format: `Bearer <token>`.
</ParamField>

## Path parameters

<ParamField path="id" type="string" required>
  UUID of the saved schedule to update.
</ParamField>

## Request body

<ParamField body="name" type="string" required>
  New display name for the saved schedule. Maximum 50 characters.
</ParamField>

## Response

Returns the updated saved schedule object.

<ResponseField name="id" type="string" required>
  UUID of the saved schedule.
</ResponseField>

<ResponseField name="name" type="string" required>
  Updated display name.
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp of when the schedule was originally saved.
</ResponseField>

<ResponseField name="sections" type="object[]" required>
  The sections that make up this schedule (unchanged by this operation).

  <Expandable title="section properties">
    <ResponseField name="sectionId" type="string" required>
      UUID of the section.
    </ResponseField>
    <ResponseField name="subject" type="string" required>
      Name of the subject.
    </ResponseField>
    <ResponseField name="professor" type="string | null" required>
      Full name of the professor, or `null` if unassigned.
    </ResponseField>
    <ResponseField name="startTime" type="string" required>
      Section start time in `HH:MM` format (24-hour, UTC).
    </ResponseField>
    <ResponseField name="endTime" type="string" required>
      Section end time in `HH:MM` format (24-hour, UTC).
    </ResponseField>
    <ResponseField name="days" type="string[]" required>
      Days of the week on which the section meets.
    </ResponseField>
  </Expandable>
</ResponseField>

## Error codes

| Status | Description |
| ------ | ----------- |
| `400`  | Validation error — `name` is empty or exceeds 50 characters. |
| `401`  | Missing or invalid Bearer token. |
| `404`  | No saved schedule with the given ID exists for the authenticated user. |

<RequestExample>

```bash cURL
curl --request PUT \
  --url https://api.scheduly.app/api/saved-schedules/1a2b3c4d-5e6f-7890-abcd-ef1234567890 \
  --header 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...' \
  --header 'Content-Type: application/json' \
  --data '{
    "name": "Updated schedule name"
  }'
```

</RequestExample>
