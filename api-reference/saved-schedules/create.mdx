---
title: 'Save a schedule'
description: 'Persist a generated schedule combination by providing a name and the section IDs from the result.'
---

`POST /api/saved-schedules`

Saves a schedule combination returned by the [generate endpoint](/api-reference/scheduling/generate). Pass the `sectionId` values from the schedule you want to keep along with a display name.

## Authentication

Requires a valid Bearer token in the `Authorization` header.

<ParamField header="Authorization" type="string" required>
  Bearer token obtained from the login endpoint. Format: `Bearer <token>`.
</ParamField>

## Request body

<ParamField body="name" type="string" required>
  Display name for the saved schedule. Maximum 50 characters.
</ParamField>

<ParamField body="sectionIds" type="string[]" required>
  Array of section UUIDs to associate with this schedule. Must contain at least one entry. Use the `sectionId` values from a previous generate response.
</ParamField>

## Response

Returns the newly created saved schedule object.

<ResponseField name="id" type="string" required>
  UUID of the saved schedule.
</ResponseField>

<ResponseField name="name" type="string" required>
  Display name of the saved schedule.
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp of when the schedule was saved.
</ResponseField>

<ResponseField name="sections" type="object[]" required>
  The sections that make up this schedule.

  <Expandable title="section properties">
    <ResponseField name="sectionId" type="string" required>
      UUID of the section.
    </ResponseField>
    <ResponseField name="subject" type="string" required>
      Name of the subject.
    </ResponseField>
    <ResponseField name="professor" type="string | null" required>
      Full name of the professor, or `null` if unassigned.
    </ResponseField>
    <ResponseField name="startTime" type="string" required>
      Section start time in `HH:MM` format (24-hour, UTC).
    </ResponseField>
    <ResponseField name="endTime" type="string" required>
      Section end time in `HH:MM` format (24-hour, UTC).
    </ResponseField>
    <ResponseField name="days" type="string[]" required>
      Days of the week on which the section meets.
    </ResponseField>
  </Expandable>
</ResponseField>

## Error codes

| Status | Description |
| ------ | ----------- |
| `400`  | Validation error — `name` is empty, exceeds 50 characters, or `sectionIds` is missing or empty. |
| `401`  | Missing or invalid Bearer token. |
