---
title: 'List saved schedules'
description: 'Retrieve all schedules saved by the authenticated user, ordered from newest to oldest.'
---

`GET /api/saved-schedules`

Returns all saved schedules that belong to the authenticated user. Results are ordered by `createdAt` descending (most recently saved first).

## Authentication

Requires a valid Bearer token in the `Authorization` header.

<ParamField header="Authorization" type="string" required>
  Bearer token obtained from the login endpoint. Format: `Bearer <token>`.
</ParamField>

## Response

Returns an array of saved schedule objects. Returns an empty array if the user has no saved schedules.

<ResponseField name="[].id" type="string" required>
  UUID of the saved schedule.
</ResponseField>

<ResponseField name="[].name" type="string" required>
  Display name of the saved schedule.
</ResponseField>

<ResponseField name="[].createdAt" type="string" required>
  ISO 8601 timestamp of when the schedule was saved.
</ResponseField>

<ResponseField name="[].sections" type="object[]" required>
  The sections that make up this schedule.

  <Expandable title="section properties">
    <ResponseField name="sectionId" type="string" required>
      UUID of the section.
    </ResponseField>
    <ResponseField name="subject" type="string" required>
      Name of the subject.
    </ResponseField>
    <ResponseField name="professor" type="string | null" required>
      Full name of the professor, or `null` if unassigned.
    </ResponseField>
    <ResponseField name="startTime" type="string" required>
      Section start time in `HH:MM` format (24-hour, UTC).
    </ResponseField>
    <ResponseField name="endTime" type="string" required>
      Section end time in `HH:MM` format (24-hour, UTC).
    </ResponseField>
    <ResponseField name="days" type="string[]" required>
      Days of the week on which the section meets.
    </ResponseField>
  </Expandable>
</ResponseField>

## Error codes

| Status | Description |
| ------ | ----------- |
| `401`  | Missing or invalid Bearer token. |

<RequestExample>

```bash cURL

  --url https://api.scheduly.app/api/saved-schedules \
  --header 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...'
```
