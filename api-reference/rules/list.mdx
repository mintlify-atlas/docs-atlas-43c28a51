---
title: 'List rules'
description: 'Retrieve all optimization rules belonging to the authenticated user, ordered by priority.'
---

`GET /api/rules`

Returns all rules belonging to the authenticated user, sorted by `priorityOrder` ascending (highest priority first).

<Info>
  All requests must include a valid Bearer token in the `Authorization` header.
</Info>

## Response

Returns an array of rule objects.

<ResponseField name="id" type="string" required>
  Unique identifier (UUID) of the rule.
</ResponseField>

<ResponseField name="type" type="string" required>
  The rule type, e.g. `"NO_GAPS"` or `"AVOID_PROFESSOR"`.
</ResponseField>

<ResponseField name="priorityOrder" type="number" required>
  Priority order. Lower values are applied first by the scheduling engine.
</ResponseField>

<ResponseField name="parameters" type="object">
  Parameters object for parameterized rules, or `null`.
</ResponseField>

<ResponseField name="userId" type="string" required>
  UUID of the user who owns this rule.
</ResponseField>

## Error codes

| Status | Description |
| ------ | ----------- |
| `401`  | Unauthorized — missing or invalid Bearer token. |

<RequestExample>

```bash cURL
curl --request GET \
  --url https://api.scheduly.app/api/rules \
  --header 'Authorization: Bearer <token>'
```

</RequestExample>
