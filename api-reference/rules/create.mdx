---
title: 'Create rule'
description: 'Create an optimization rule that the scheduling engine uses to score and rank generated schedules.'
---

`POST /api/rules`

Creates a new scheduling optimization rule for the authenticated user. Rules are applied by the scheduling engine when generating schedule suggestions, ordered by `priorityOrder` (lower value = higher priority).

<Info>
  All requests must include a valid Bearer token in the `Authorization` header.
</Info>

## Request body

<ParamField body="type" type="string" required>
  The rule type. Must be one of the 11 supported rule types listed below.

  | Value | Description | Requires parameters |
  | ----- | ----------- | ------------------- |
  | `NO_GAPS` | Avoids gaps between back-to-back classes on the same day. | No |
  | `NO_EARLY_MORNINGS` | Avoids sections that start before 08:00. | No |
  | `NO_LATE_EVENINGS` | Avoids sections that end after 18:00. | No |
  | `PREFER_MORNING` | Rewards sections scheduled between 06:00 and 14:00. | No |
  | `PREFER_AFTERNOON` | Rewards sections scheduled between 14:00 and 22:00. | No |
  | `COMPACT_DAYS` | Rewards schedules that use fewer distinct days. | No |
  | `BALANCED_LOAD` | Rewards even distribution of class hours across days. | No |
  | `MIN_FREE_DAY` | Rewards having at least one completely free day per week. | Yes — day selection |
  | `MAX_CONSECUTIVE_HOURS` | Penalizes consecutive class blocks longer than the given threshold. | Yes — number |
  | `LUNCH_BREAK_PROTECTED` | Excludes sections that overlap a protected lunch time range. | Yes — time range |
  | `AVOID_PROFESSOR` | Excludes all sections taught by the specified professor. | Yes — professor UUID |
</ParamField>

<ParamField body="priorityOrder" type="number" required>
  Priority of this rule relative to others. Lower values are evaluated first. Minimum value: `1`.
</ParamField>

<ParamField body="parameters" type="object">
  Additional configuration required by parameterized rule types. Omit for rules that take no parameters.

  **`MIN_FREE_DAY`** — specify a preferred free day:
  ```json
  { "type": "daySelect", "value": 5 }
  ```
  `value` is a day ID from 1 (Monday) to 7 (Sunday).

  **`MAX_CONSECUTIVE_HOURS`** — maximum consecutive hours before a break:
  ```json
  { "type": "number", "value": 3 }
  ```
  `value` is an integer from 1 to 8.

  **`LUNCH_BREAK_PROTECTED`** — protected time window for lunch:
  ```json
  { "type": "timeRange", "value": { "start": "12:00", "end": "14:00" } }
  ```
  `start` and `end` are in `HH:MM` 24-hour format.

  **`AVOID_PROFESSOR`** — professor to exclude from all sections:
  ```json
  { "type": "professorSelect", "value": "b2c3d4e5-f6a7-8901-bcde-f12345678901" }
  ```
  `value` is the UUID of the professor.
</ParamField>

## Response

<ResponseField name="id" type="string" required>
  Unique identifier (UUID) of the created rule.
</ResponseField>

<ResponseField name="type" type="string" required>
  The rule type string, e.g. `"NO_GAPS"`.
</ResponseField>

<ResponseField name="priorityOrder" type="number" required>
  Priority order of the rule.
</ResponseField>

<ResponseField name="parameters" type="object">
  The stored parameters object, or `null` for parameter-free rules.
</ResponseField>

<ResponseField name="userId" type="string" required>
  UUID of the user who owns this rule.
</ResponseField>

## Error codes

| Status | Description |
| ------ | ----------- |
| `400`  | Validation error — missing or malformed fields. |
| `401`  | Unauthorized — missing or invalid Bearer token. |

<RequestExample>

```bash cURL (no parameters)
curl --request POST \
  --url https://api.scheduly.app/api/rules \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "type": "NO_GAPS",
    "priorityOrder": 1
  }'
```

```bash cURL (LUNCH_BREAK_PROTECTED)
curl --request POST \
  --url https://api.scheduly.app/api/rules \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "type": "LUNCH_BREAK_PROTECTED",
    "priorityOrder": 2,
    "parameters": { "type": "timeRange", "value": { "start": "12:00", "end": "14:00" } }
  }'
```

```bash cURL (AVOID_PROFESSOR)
curl --request POST \
  --url https://api.scheduly.app/api/rules \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "type": "AVOID_PROFESSOR",
    "priorityOrder": 3,
    "parameters": { "type": "professorSelect", "value": "b2c3d4e5-f6a7-8901-bcde-f12345678901" }
  }'
```

</RequestExample>
