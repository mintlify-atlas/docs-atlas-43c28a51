# Scheduly API

Scheduly API is a NestJS REST API that generates optimized academic schedules for university students. It uses a backtracking algorithm combined with a priority-based rule scoring system to find the best schedule combinations based on your preferences.

## What Scheduly API does

University students often struggle to build optimal schedules among dozens of section options for each subject. Scheduly API solves this by:

1. Accepting the subjects you want to enroll in
2. Applying your personal optimization rules (avoid early mornings, prefer compact days, etc.)
3. Running a backtracking algorithm to find all conflict-free combinations
4. Scoring each combination by your rules ranked by priority
5. Returning the **top 3 best schedule combinations**

## How it works

The scheduling engine works in two phases:

**Phase 1 — Pruning**: Filters out sections that violate absolute rules before the search begins. For example, if you have an `AVOID_PROFESSOR` rule for a specific professor or a `LUNCH_BREAK_PROTECTED` rule, sections that conflict are removed entirely.

**Phase 2 — Backtracking + Scoring**: Explores all valid combinations without time conflicts. Each valid combination receives a numeric score based on your rules (ordered by priority). The engine returns the top 3 highest-scoring combinations.

## Usage flow

<Steps>
  <Step title="Register and authenticate">
    Create an account and log in to receive a JWT access token.
  </Step>
  <Step title="Create your academic data">
    Add subjects, professors, and sections (with schedules and days).
  </Step>
  <Step title="Configure optimization rules">
    Set rules with priority order to express your scheduling preferences.
  </Step>
  <Step title="Generate schedules">
    Call `POST /api/scheduling/generate` with your subject IDs.
  </Step>
  <Step title="Save your preferred schedule">
    Use `POST /api/saved-schedules` to persist the combination you like best.
  </Step>
</Steps>
